# 餐饮点餐与优惠活动小程序 PRD

## 1. 产品定位

为窑烤披萨店提供一个微信小程序，用于顾客扫码点餐、自取下单、查看优惠活动、领取优惠券，并让门店通过后台管理菜单、订单、桌号和活动。

第一阶段目标不是做复杂会员系统，而是先解决三个经营问题：

- 顾客点餐更顺，不必排队口头下单
- 后厨订单更清楚，减少漏单和错单
- 优惠活动可配置，方便拉新和提升复购

## 2. 用户角色

| 角色 | 主要需求 |
| --- | --- |
| 顾客 | 看菜单、选规格、加购物车、用券、微信支付、查订单 |
| 店员 | 查看订单、确认制作、标记完成、核销自取 |
| 店长 | 管理菜品、价格、库存、活动、桌号和销售数据 |

## 3. 业务场景

### 堂食扫码点餐

顾客扫描桌面二维码，进入对应桌号的菜单页，选择披萨、小食和饮品，提交订单并支付。后厨看到桌号和菜品明细，制作完成后由服务员送餐。

### 到店自取

顾客在小程序中选择自取时间，下单支付。后厨按时间准备，顾客到店后凭取餐码核销。

### 优惠活动

顾客进入活动页领取优惠券，或在结算时自动匹配满减、套餐优惠。店长可以在后台设置活动时间、使用门槛、适用菜品和发券数量。

## 4. 顾客端页面

### 首页

核心内容：

- 门店状态：营业中、休息中、可自取、可堂食
- 入口：扫码点餐、自取点餐、优惠活动、我的订单
- 推荐商品：招牌披萨、套餐、新品
- 当前可用优惠

### 菜单页

核心内容：

- 分类：披萨、小食、沙拉、饮品、套餐
- 菜品列表：图片、名称、简介、价格、销量、售罄状态
- 规格选择：尺寸、饼底、辣度、加料、备注
- 加入购物车

### 购物车

核心内容：

- 已选商品
- 数量调整
- 规格展示
- 优惠预估
- 清空购物车

### 结算页

核心内容：

- 就餐方式：堂食、自取
- 桌号或自取时间
- 优惠券选择
- 订单备注
- 金额明细：商品金额、优惠金额、实付金额
- 微信支付

### 订单详情

核心内容：

- 订单状态：待支付、已支付、制作中、待取餐、已完成、已取消
- 菜品明细
- 桌号或取餐码
- 预计出餐时间
- 联系门店

### 优惠活动页

核心内容：

- 可领取优惠券
- 限时套餐
- 新品活动
- 满减说明
- 已领取优惠券

### 我的

核心内容：

- 手机号授权
- 我的订单
- 我的优惠券
- 积分或会员等级，第二阶段再做
- 联系门店

## 5. 商家后台功能

### 菜品管理

- 新增、编辑、删除菜品
- 设置分类
- 上传图片
- 设置价格
- 设置规格和加料
- 设置库存或售罄
- 设置推荐菜

### 订单管理

- 新订单提醒
- 按状态筛选订单
- 查看桌号、取餐时间和备注
- 标记制作中
- 标记已完成
- 取消或退款记录

### 桌号管理

- 新建桌号
- 生成桌码
- 启用或停用桌码
- 绑定门店

### 优惠活动管理

- 优惠券：满减券、折扣券、新客券
- 满减活动：满多少减多少
- 套餐活动：披萨 + 饮品 + 小食
- 限时活动：指定时间段生效
- 发放数量和领取限制

### 数据看板

- 今日销售额
- 今日订单数
- 客单价
- 热销菜品
- 优惠核销金额
- 堂食和自取占比

## 6. 优惠规则

第一版建议支持以下三类，规则清楚，开发难度可控：

| 类型 | 示例 | 适用场景 |
| --- | --- | --- |
| 满减券 | 满 99 减 15 | 提升客单价 |
| 新客券 | 首单减 10 | 拉新 |
| 套餐价 | 双人披萨套餐 128 | 午市、晚市、周末 |

规则优先级建议：

1. 套餐价先计算
2. 再计算满减券
3. 单笔订单只允许使用一张券
4. 特价商品是否参与满减由后台设置

## 7. 订单状态流转

```text
待支付 -> 已支付 -> 制作中 -> 待取餐/待上桌 -> 已完成
       -> 已取消
```

退款状态单独记录：

```text
无退款 / 退款中 / 已退款 / 退款失败
```

## 9. 微信能力依赖

- 微信登录：识别用户身份
- 手机号授权：用于订单联系，可选
- 微信支付：完成小程序内支付
- 订阅消息：订单状态提醒，需用户授权
- 小程序码或二维码：桌号点餐入口

## 10. 第一版范围

必须做：

- 顾客端菜单、购物车、结算、支付、订单详情
- 堂食桌码
- 自取下单
- 后台菜品管理
- 商品上架、下架、售罄和价格维护
- 后台订单管理
- 满减券或新客券
- 基础数据看板

暂缓：

- 复杂会员等级
- 储值卡
- 拼团秒杀
- 多门店连锁
- 第三方外卖平台打通
- 智能推荐

## 11. 验收标准

- 顾客能从扫码到支付完成闭环
- 后台能实时看到订单
- 店员能改变订单状态
- 菜品能上下架
- 售罄菜品不可下单
- 优惠券计算准确
- 支付成功后不会重复下单
- 网络异常时订单状态可恢复

## 12. 主要风险

- 微信支付和商户号配置需要提前准备
- 高峰期订单提醒必须稳定
- 优惠规则不能过度复杂，否则容易算错
- 菜品规格和加料要设计清楚，否则披萨类商品容易出错
- 堂食、自取、外卖混在一起时，后厨排单需要清晰区分
