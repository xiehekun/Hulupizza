# 运行环境

这个项目包含两部分：

- 经营文档：Markdown 文件，直接阅读即可。
- 微信点餐小程序 MVP：位于 `miniprogram/app`，需要用微信开发者工具运行。

## 必需工具

- Node.js 18 或以上：用于本地项目检查。
- 微信开发者工具：用于运行和调试小程序。
- Git：用于提交和推送代码。

## 本地检查

在仓库根目录运行：

```bash
node scripts/check-miniprogram.js
```

这个检查会确认：

- 小程序关键配置文件存在
- `app.json` 中声明的页面文件齐全
- `project.config.json` 类型正确
- JavaScript 文件没有基础语法错误

如果本机安装了 npm，也可以运行：

```bash
npm run check
```

## 运行小程序

1. 打开微信开发者工具。
2. 选择“导入项目”。
3. 项目目录选择：

```text
miniprogram/app
```

4. AppID 可以先使用测试号；有正式小程序后，把 `project.config.json` 里的 `appid` 改成正式 AppID。
5. 点击“编译”运行。

## 当前 MVP 使用方式

- 首页选择“堂食点餐”或“到店自取”
- 进入菜单选择商品和规格
- 加入购物车
- 到优惠活动页领取券
- 结算页选择优惠券并模拟支付
- 订单详情页模拟出餐完成

## 后续接入真实环境

要变成可上线版本，还需要：

- 微信小程序正式 AppID
- 微信支付商户号
- 服务端接口
- 数据库
- 后厨或商家后台
- 真实桌号二维码

第一阶段可以继续在本地缓存里迭代界面和流程，等流程稳定后再接后端。
